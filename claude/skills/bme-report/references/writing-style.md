# 文体ガイド（下地慶英の書き方）

過去レポートから抽出した文体の特徴と実例。**AIっぽさを排除し、下地本人の言い回しを再現する**ためのファイル。
本文を書くときは、この特徴とサンプルに寄せる。

## 1. 基本トーン

- **だ・である調**で統一。です・ます調は使わない。
- **実験方法は過去形**（「〜した」「〜を確認した」）。理論・考察は現在形・推量が中心。
- 淡々と、事実と理屈を積み上げる。誇張・感嘆・比喩は使わない。
- 一文はやや長め。「〜であり、〜のため、〜となる。」のように理由や条件をつないで書く。
- 補足は半角括弧 `(TV)` `(Lesson12)` `(コンパレータ)`、単位は `[s]` `[mV]` `[deg]`。

## 2. 文末・語尾のパターン

よく使う結び：
- `〜である。` / `〜となる。` / `〜となった。`
- `〜を目的とする。`（目的の結び）
- `〜を以下に示す。` / `〜を表◯に示す。` / `〜を図◯に示す。`
- `〜のようである。` / `〜のようになる。` / `〜のとおりである。`
- `〜と考えられる。` / `〜と推測される。` / `〜と示唆する。`（考察の結び）
- `〜が原因として考えられる。` / `〜が要因として挙げられる。`
- `〜に注意する。` / `〜する必要がある。` / `〜することとする。`（手順・注意）
- `〜が求められる。` / `〜が可能である。`

`したがって、` `ゆえに、` `ここで` `具体的には、` `以上より、` `これにより、` `逆に、`

## 4. 定型フレーズ（章別）

- 冒頭の主語：`本実験では` / `本実習では` / `本課題では`
- 目的：`〜を理解することを目的とする。` / `〜が実験の目的である。` — **目的は資料の文言をほぼ踏襲**し、
  必要なら「加えて、〜を合わせて理解する。」と1文足す程度。自分で大きく書き換えない。
- 章見出し直後のリード文：`本実験で用いる基本的な原理についてまとめる` /
  `本実験で使用した器具は以下のとおりである。` / `本実験のレポート課題を以下に述べる。` /
  `本実験で行った5つの実験の内容についてまとめる。`
- 理論導入：`以下、〜について記す。` / `〜の原理について述べる。`
- 装置：`本実験で使用した実験装置は以下のとおりである。` / `〜を用いた。`
- 方法：`以下の手順で計測を実施した。` / `〜した後、〜を確認した。`
- 結果：`表◯に計測結果と計算結果をまとめた。` / `〜が得られた。`
- 考察：`〜の結果について考察する。` / `この結果から、〜と考えられる。`
- 利点欠点を並べるときは `【利点】` `【欠点】` の見出し＋箇条書き。

## 6. AIっぽさを避けるルール

- **禁止**：「本レポートでは〜について詳しく見ていきます」「〜が重要です」「まとめると」「いかがでしたか」的なメタ表現・語りかけ。
- **禁止**：箇条書きの多用でごまかす（説明すべき所は文章で理由まで書く）。
- **禁止**：均等に整いすぎた対句・過剰な太字強調・絵文字。
- 抽象論で終わらせず、**具体的な数値・型番・条件**を必ず入れる（例：`+1.00mmの誤差`、`r = 0.9922`、`SS11LA`）。
- 因果を省略しない。「〜なので〜となる」まで書く。下地の文章は理由の連鎖が長い。
- 過度に丁寧・冗長な前置きを付けない。淡々と本題に入る。
- 同じ語尾の連続はある程度許容（本人の文章もそうなっている）。無理に言い換えて不自然にしない。
- **人間の軽微な揺れ・誤記を過剰に矯正しない**（キャプションの微妙な重複、語の言い間違い等）。整いすぎるとAIっぽくなる。
- **考察は資料の問いに答えるだけで終わらせず、自発的に一段深掘りする**（追加検証・既存ツールとの実測比較・初期実装との対比など）。抽象的な調査で締めず、自分で数値を出す。
