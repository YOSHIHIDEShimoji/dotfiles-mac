## 1. 表紙

1ページ目は表紙。すべて **ＭＳ ゴシック**。表紙のあとは改ページして本文へ。
値の入る行には**下線**が付き、項目間には空行が多め（build_report.py の cover が自動でやる）。

| 行 | 内容例 | サイズ | 揃え |
|---|---|---|---|
| レポート題 | `医工学実験Ⅱ　レポート` | 24pt 太字 | 中央 |
| 実験題目 | `実験題目　　○○について` | 16pt 下線 | 左 |
| 実験日 | `実験日　　2026年　○月　○日` | 16pt 下線 | 中央 |
| 提出日 | `提出日　　2026年　○月　○日` | 16pt 下線 | 中央 |
| 学生番号 | `医工学コース　学生証番号　　　24TB4039` | 16pt 下線 | 中央 |
| 氏名 | `氏名　　　下地　慶英` | 16pt 下線 | 中央 |
| 共同実験者 | `共同実験者　　○○　○○（学籍番号）` （複数なら行を分ける） | 16pt 下線 | 中央 |

- 医工学実験の番号は資料に従う（Ⅰ/Ⅱ/Ⅲ）。ラベルと値の間は全角スペースで揃える。
- 実験日・提出日が不明なときは `2026年　　　月　　　日` と空欄にする（実験前モード）。
- 共同実験者は氏名＋（学籍番号）まで書くことが多い。

## 2. 見出し階層とフォント

本文は **ＭＳ 明朝**。見出しは Word 標準スタイル（Heading 1/2/3）で、太字。

| レベル | スタイル | サイズ | 使い方 |
|---|---|---|---|
| 見出し1 | Heading 1 | 16pt 太字 | 章（目的、理論、実験装置…） |
| 見出し2 | Heading 2 | 14pt 太字 | 節（2.1, 2.2…） |
| 見出し3 | Heading 3 | 12pt 太字 | 項（2.1.1…／手順の小分け） |

- **番号は手動で見出しテキストに含める**（自動採番ではない）。例 `2. 理論`、`2.1 PID制御`、`4.1.1 接続とキャリブレーション`。
- 章番号は `1.` のようにドット付きが最頻。節・項は `2.1` `2.1.1`。
- 見出し1の直後に見出し2、その下に本文…という順序を守る。空の見出しは作らない。

## 3. 章立て（実験タイプ別）

資料の内容から実験タイプを判定し、対応する章立てを土台にする。**資料の目次・課題構成があればそれを最優先**で反映する。

### A. 計測・回路系の実験（biopac, アナログ回路, 自動制御, 機械加工 等）
```
1. 目的
2. 理論（原理）          ← 資料の理論・原理を自分で纏め直す。複数テーマなら 2.1, 2.2…
3. 実験装置              ← 製品名・型番・個数まで明記
4. 実験方法              ← 実施した手順を過去形で。テーマ別に 4.1, 4.2…
5. 実験結果              ← 図表中心。事実のみ
6. 考察                  ← 結果から言えること。レポート課題があればここか別章
（7. レポート課題）       ← 資料に課題があれば独立章にすることも多い
最終章. 参考文献
```

### B. プログラミング系の実験（C1〜C5）
```
1. 実験テーマ全体としての学習の目的  ← 目的は資料踏襲でなく「〜できるようになることである」と学習目的として再構成してよい
2. 課題内容              ← 2.1 課題1, 2.2 課題2 …（各課題の説明＋その課題で使う理論・数式）
（3. プログラムソース）  ← 資料に「プログラムソース」等の記載があれば独立章で全課題分を貼る。
                           コードは code ブロック(等幅・罫線枠)で。付録Gistと二重掲載するのが本人の型。
                           「貼付不要」と資料にあれば置かず付録Gistのみ
次章. 実行結果           ← 課題別。実行結果は図でなく**コンソール出力の逐語貼付**(code ブロック=等幅枠)が基本。
                           「実行コマンドを以下に示す」+枠、「出力結果を以下に示す」+枠 のペア構成
次章. 考察（検討および考察）← 課題別に分割。資料要求の穴埋めで終わらせず一段深掘りする（下記）
次章. 全体の考察とまとめ   ← 課題別考察の後に総括章を置くのが最近の型
次章. 意見・感想          ← 準必須。独立章で常設する
次章. 参考文献            ← 本文で [n] を引用したら必ず置く
最終章. 付録              ← 「本実験で使用したデータファイル、C言語ソースコードは以下のとおりである。
                           すべてのファイルは次の GitHub Gist から入手可能である。」+ Gist URL(中央) +
                           表「本実験で使用したプログラムおよびデータ一覧」(ファイル名|説明)
```
- 課題の理論・数式（DFTの定義式など）は**課題内容の各節に**書く（独立した理論章は作らない）。
- 数式が多い場合、複雑な数式（総和記号・分数など）はプレーンテキストでは崩れるため、近似表記で置いた上で「Wordの数式機能で清書する」赤注記を添える。
- **考察の深掘り**（下地の水準）：資料が求める考察項目を骨格に、さらに自発的な追加検証を自作図で上乗せする。例：printf有無の速度比較、コンパイラ最適化オプション(-O0/-O2/-O3)比較、メモリ使用量比較、既存ライブラリ(OR-Tools等)との実測比較、初期実装→改善実装の対比。結果章の図＝資料要求分、考察章の図＝主張を裏づける追加分、の二層になる。
- **計算量・計算時間系の締め**：モデル式→大きなNの予測→年換算→宇宙年齢(138億年)との比較で不可能性を断ずる、が下地の定型。
- コード提出対象でない課題（モデル化のみ等）はソース章・実行結果章の対象から外す。
- 実データと掲載データが違う場合は注記する（例「実データはN=30だが冗長なのでN=10で記載する」）。

### D. 調査・課題応答型（座学レポート：製図の公差回など）
装置・測定・結果・考察の章が無く、資料が「調べて図を用いてまとめよ」と問い(Q)を並べる実験。
判定ヒント：講義資料の大半が一般論・記法で、課題スライドが独立に問いを列挙し、報告課題の理論を資料が与えていない（＝各自調査）なら D型。
```
（目的章すら置かないことが多い。いきなり課題1本文から始まる）
1. <課題1の要約見出し>   ← 定義 → 具体例(数値計算/図) 
2. <課題2の要約見出し>   ← 比較・分類など
…                        ← 課題番号=章番号で1対1
最終章. 参考文献
```
- 見出しは課題原文そのままでなく**自分の要約見出し**に言い換えてよい。ただし課題番号と1対1対応は厳守。
- 「具体的な数字を用いて」「○ページ以内」等の制約を守る。**数値計算例は空欄で残さず実際に計算値を埋める**。
- 資料が理論を与えないので、外部調査＋自分の言葉で書く。丸写ししない。
- 出典は個人ブログ・企業コラムに加え、可能な限り **JIS/ISO の規格番号**（JIS Z 8103、JIS B0601、ISO25178 等）を併記する。

### C. 演習課題＋レポート課題型（ディジタル回路 等、資料が「実験課題」「レポート課題」を分けて指定する実験）
```
1. 目的
2. 原理                  ← 資料の理論を項目ごとにフル展開（下記「原理章の書き方」）
3. 使用器具              ← 「本実験で使用した器具は以下のとおりである。」+ 中黒箇条書き
4. 実験課題              ← 4.1 実験1 … 各実験は課題文の要約 + 実施手順の過去形箇条書き
                           （(1)真理値表を作成した。(2)論理式を作成した。…）だけ。結果は書かない
5. レポート課題          ← 資料の「レポート課題」の番号にそのまま対応させる
   5.1 実験課題のまとめ   ← 5.1.1 実験1, 5.1.2 実験2… に分け、
                           各実験の真理値表・式・図・写真・接続図など「成果物」を全部ここに置く
   5.2, 5.3 …            ← 残りのレポート課題
6. 考察・感想
7. 参考文献
```
**重要**: 資料に「レポート課題」の指定がある場合は、その番号・順序に章立てを厳密に対応させる。
「実験課題の◯題についてまとめよ」という課題があるなら、真理値表・論理式・結果は
実験課題の章ではなく**レポート課題の章**に置く。実験課題の章は手順の過去形要約に留める。

- 複数テーマ／課題がある実験では、**理論・方法・結果・考察すべてを同じテーマ分割（○.1, ○.2…）で並行させる**のが下地の型（例：biopac は全章を Lesson 単位で分割）。目的・理論・装置・方法・結果・考察の全章を同一の下位分割（○.1=Lesson16, ○.2=Lesson17 等）で貫く。
- **考察は結果より一段細かく刻んでよい**（結果 5.1 に対し考察 6.1.1 / 6.1.2… を `[3]`=h3 で細分）。回路別・論点別に分割する。
- **複数テーマ共通の理論（t検定・統計処理など）は、各テーマの理論節より前に独立節（例「2.0 t検定」）として置く**。テーマ別分割の前に共通理論ブロックを設ける。
- 「感想」「意見・感想」「本実験を通して理解したこと」を末尾付近に置くことが多い。
- 章見出しの直後に1文のリード文を置く癖がある（「本実験で用いる基本的な原理についてまとめる」「本実験のレポート課題を以下に述べる。」「本実験で使用した器具は以下のとおりである。」）。
- **見出しの番号運用には幅がある**：章までドット付き通し番号（`1.` `2.1`）が最頻だが、章見出しに番号を付けず節・図表・式番号だけ章番号で運用する折衷型もある。**同一レポート内では一貫**させる。
- **レポート課題は資料の課題番号・順序に厳密対応**（7.1, 7.2…）。これは C/D 型だけでなく計測系のレポート課題章にも適用する。

### 原理章の書き方（タイプA/C共通）

資料の理論を**文章の要約だけで済ませない**。項目ごとに小見出し（節、または①②③の丸数字）を立て、
各項目に対して：
- 定義・動作の説明（1〜3文）
- 論理式・数式（式番号付き、`eq` ブロック）
- 図（論理図など。実験前モードでは `figure` プレースホルダ）
- 真理値表などの表（導出できるなら `table_data` で実表を組む）
- 対応する具体物（使用ICの型番など）があれば明記
をセットで並べる。例：ディジタル回路の原理章では AND/OR/NOT/NAND/NOR/XOR をそれぞれ
「①AND（論理積）… 論理式は式2-1に、論理図、真理値表はそれぞれ図2-1、表2-1に示す。74シリーズICでは、74HC08に該当する。」
のように①〜⑥で展開する。

## 4. 図のルール（block type = `figure` / `fig_done`）

- **キャプションは画像の下**・中央揃え。Caption スタイル（太字）。
- 番号形式は `図 <章>-<連番>`。**図と番号の間に半角スペース、番号と図名の間に全角スペース**（例 `図 2-1　ANDの論理図`）。この形を必ず守る（`図21` や `図 2-1論理図` のようにハイフン・全角スペースを落とさない。caption 文字列は必ず `f"図 {章}-{連番}　{図名}"` の形で組む）。
- **番号は章ごとにリセットして採番**（図3-1=方法, 図4-1=結果, 図5-1=考察）。資料の元図番号をそのまま流用しない。※ただし一部の実験（biopac1等）では図と表で連番を共有し章内出現順に振る流儀もある。**その実験の過去例・資料に合わせ、レポート内で一貫**させる。
- 図・画像本体も中央揃え。
- 自分で作成した図でない場合は末尾に出典 `[n]` を付ける。
- グラフは：白黒印刷を想定し実線/破線＋凡例で区別、縦軸・横軸のラベルと**単位**、スケールを明記。実験値はプロット、理論値は曲線。
- **グラフ化条件が資料にある場合（「平均でなく個々のデータ」「全結果を同一グラフに」「主軸/第2軸の割当」等）は、プレースホルダ／キャプションにその条件を転記**する。
- 実験前モード：`figure` で赤プレースホルダ（画像位置）＋その下に黒のキャプション。プレースホルダには「何のグラフか・軸・単位」まで具体的に書く。
- **1条件を複数図に分けることを許容/推奨**（正ピーク図と負ピーク図、波形図とスペクトル図など）。成果物カウントでは条件別・正負別で図が増えることを見込む。

## 5. 表のルール（block type = `table` / `table_data`）

- **キャプションは表の上**・中央揃え。Caption スタイル（太字）。
- 番号形式は `表 <章>-<連番>`。図と同様、`表 2-1　ANDの真理値表` のようにスペースを入れる。
- 表本体も中央揃え。自分で作成したもの以外は出典明記。
- **テキストで組む表（真理値表など）は三線表**（横罫線のみ。上端・ヘッダ下・下端。縦罫線なし）。
  `table_data` がこのスタイルで出力する。真理値表では `group_header: [["入力", N], ["出力", M]]`
  で「入力／出力」のグループ見出し行を付けるのが本人の型。
- Excel で作ったグラフ・集計表は画像として貼ることが多い（その場合は `figure`/`fig_done` と同じ扱いで、ただしキャプションは上）。
- **資料に「データシートをそのまま添付せず各自作成し直す」指示がある場合は、結果表を `table_data` で再構築**する（画像貼付で済ませない）。
- 実験前モード：`table` でキャプション（上）＋その下に赤プレースホルダ。プレースホルダに「どんな列・行のデータか」を具体的に書く。

### ソースコード・実行結果（block type = `code`、プログラミング系）
- ソースコード・実行コマンド・コンソール出力は `code` ブロック（等幅Menlo・罫線枠の1セル表）で貼る。
- 「〜を以下に示す。」というリード文（`lead`）＋枠 のペアで構成する。

## 6. 数式のルール（block type = `eq`）

- 数式は独立した行、**本文中央に数式・右端に式番号**。
- 式番号の形式は `式 <章>-<連番>`（例 `式 2-1`）。`num` に "式 2-1" を渡す。
- 数式中で使う文字は、直後の本文または箇条書きで必ず説明する。
- 本文からの参照は「論理式は式 2-1 に示す」「式(1)で与えられる」のように必ず番号で参照する。

## 7. 本文の書式

- 文体は **だ・である調**。→ 文体の詳細と例文は `writing-style.md` を参照。
- **実験方法は過去形**（「〜した」「〜を確認した」）。レポートは報告書だから。
- 指導書の丸写しにしない。実施した手順を、実験ごとに纏め直して書く。「○○を参照する」で済ませない。
- 段落の頭は全角1字下げ（`p` ブロックが自動で付与）。
- 引用・参考にした箇所は本文中に `[n]` を付け、末尾の参考文献と対応させる。

### 実験装置の書き方（補強）
- 型番・個数だけでなく、**シリアル番号(SN)・機体識別番号**まで書く（例 `呼吸流量トランスデューサ SS11LA (SN901 6494)`、`hp製PC(BIOPAC #4)`）。
- **会社名・代理店名は資料から正確に転記**（例 BIOPAC製品の国内代理店は「株式会社モンテシステム」。`System社製` のような略記・誤記をしない）。
- 選択した抵抗値などの**理論計算（式からの導出）は結果章でなく実験方法章の該当手順に書く**（「式(x)により理論値は○kΩだが、入手可能な○kΩを選定した」）。

## 8. 参考文献（最終章）

- 章見出し `参考文献`（または `引用文献`）。`[1] 著者, "題名," 媒体, URL, 参照日：…` の形式。
- URL のみの記載は不可（何のサイトか明示）。Wikipedia は不可。可能なら JIS/ISO の規格番号を併記。
- 本文で引用した箇所と番号を対応させる。**本文で `[n]` を使ったら必ず参考文献章を置く**（省略しない）。本文未引用の番号を残さない。

## 9. spec 生成のチェックリスト

spec を組み立てる前に必ず確認：

**構成**
- [ ] 実験タイプ（A計測/B プログラミング/C 演習+課題/D 調査応答）を判定し、**資料の目次・構成例・レポート課題指定を最優先**で土台にしたか（資料の最終ページに構成例が付くことがある）
- [ ] 資料に複数テーマ/課題があるなら、全章を同じ分割で並行させたか。共通理論は前に独立節を置いたか
- [ ] レポート課題・必須考察項目があれば、資料の番号・項目に**1対1で対応**させたか（生成後に本文がすべての項目に正対しているか検算）

**成果物の数え上げ（最重要）**
- [ ] 資料の「記録/グラフ化/図示/表化/考察/描く/提出」指示をすべて拾い、図・表を1つ残らず数えたか
- [ ] パラメータ掃引は**値の数**、散布図は**条件数+全体(1)**、回路は**回路数×条件数**で件数を計算したか
- [ ] t検定・相関係数r の指定があれば、それに伴う**散布図＋回帰直線を条件ごとに**用意したか
- [ ] 各成果物を個別のプレースホルダ（番号・キャプション・軸・単位入り）にしたか（「以降同様」で省略していないか）
