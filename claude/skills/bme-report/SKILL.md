# 医工学実験レポート生成 (bme-report)

実験資料のディレクトリから、下地慶英の過去レポートに揃えた .docx レポートを生成する。

## 前提

- docx 生成に `python-docx` が必要。専用の仮想環境がある：
  `~/.pyenv/versions/bme-report-3.11.9/bin/python`（無ければ `pyenv virtualenv 3.11.9 bme-report-3.11.9` を作り `pip install python-docx`）。
- スクリプト：`scripts/build_report.py`（spec.json → docx）。同梱テンプレート `assets/template.docx` からスタイル（Heading 1-3=16/14/12pt太字, Caption, List Paragraph 等）を継承する。
- 書式ルール：`references/report-format.md`。文体：`references/writing-style.md`。**本文を書く前に必ず両方を読む。**

## ワークフロー

### 手順1：実験書を精読してモードを決める

渡されたディレクトリ（1実験=1ディレクトリ）の中身を確認する。

- **資料（指導書・実験説明書・指針・理論PDF、装置写真等）を全ページ精読する**。要点の拾い読みで済ませない。実験を実際にやる人間が読むのと同じ深さで読む。
- **測定データ・結果グラフ・実験写真** があるか確認する。
  - なければ → **実験前モード**（雛形生成）。
  - あれば → ユーザーに「実験後モードで結果・考察まで作るか」を確認し、**実験後モード**。
- 指示があればそれに従う（「雛形だけでいい」等）。

**精読しながら「成果物リスト」を作る（最重要）**。資料中の次の表現をすべて拾い、レポートに載る図・表・記録を1つ残らず数え上げる：
- 「記録する」「ログを取得する」「グラフ化する」「図に示しなさい」「表にまとめなさい」「考察しなさい」「描きなさい」「提出する」
- パラメータを振る実験は**値の数だけ**成果物がある（例：ゲインを350から500まで刻み幅50で変化→350/400/450/500の4本のログ＝4枚のグラフ）。刻み幅から本数を計算する。
- 資料が指定するファイル名規則（例 `BodyAngururSPD_-4000`）、グラフの軸・単位・種別（散布図(直線)等）の指定もメモする。

この成果物リストの件数と、生成する図・表プレースホルダの件数を**一致**させる。「図5-2以降を同様に貼る」のような一括プレースホルダで省略しない — 1成果物=1プレースホルダで、番号・キャプション・軸まで個別に書く。

### 手順2：構成を設計する

`references/report-format.md` の章立て4タイプ（A:計測・回路系 / B:プログラミング系 / C:演習課題+レポート課題 / D:調査・課題応答型）と、**資料の目次・構成例・レポート課題指定を最優先**で突き合わせて章立てを決める（資料の最終ページに構成例が付くことがある）。複数テーマ/課題があれば、理論・方法・結果・考察を同じ分割で並行させ、共通理論は前に独立節を置く（下地の型）。

### 手順3：本文を書く（spec を組み立てる）

`references/writing-style.md` に沿って、下地の文体で本文を書く。

- **黒字（完成させる）**：目的・理論・実験装置・実験方法。資料から書ける範囲を書き切る。理論は丸写しせず自分の言葉で纏め、数式は式番号＋文字説明を付ける。方法は過去形。**資料に値が載っている表（装置の仕様表・パラメータ初期値表・真理値表など）は `table_data` で実データの表を組む**（プレースホルダにしない）。
- **赤字（ユーザーが埋める）＝実験前モード**：結果・考察・図・表。`p_red` / `figure` / `table` ブロックで、**何を入れるか具体的に指示**した赤プレースホルダにする（例『［図5-1をここに挿入：本体角度の時間変化(横軸:時間[sec],縦軸:本体角度[deg])］』『［予測：数値を大きくした場合の挙動とその理由］』）。
- **実験後モード**：結果は取得データ・図から事実を記述し、`fig_done` で実画像を貼る（どの図をどこに入れるかユーザーに確認）。考察は結果に基づき下地の文体で完成させる。

spec は JSON。ブロック type（h1/h2/h3, p, b, p_red, ul, ul_red, eq, figure, fig_done, table, table_data, code, spacer）と cover の形式は `scripts/build_report.py` 冒頭の docstring を参照。資料に値が載っている表（装置仕様・パラメータ初期値・真理値表）は `table_data` で実表を組む。プログラミング系のソース・実行結果は `code`（等幅・罫線枠）で貼る。

**成果物の数え上げ（手順1で作ったリストを反映）**：図・表を1つ残らず個別プレースホルダにする。パラメータ掃引は値の数、散布図は条件数+全体、t検定/相関指定は条件ごとの散布図＋回帰直線。「以降同様」で省略しない。

### 手順4：生成する

spec を JSON で書き出し、実行：
```bash
~/.pyenv/versions/bme-report-3.11.9/bin/python \
  scripts/build_report.py <spec.json> <出力先.docx> [--overwrite]
```
出力先は資料ディレクトリ内の**新規ファイル名**（例 `.../自動制御/レポート雛形_自動制御.docx`）。

**安全ルール（厳守）**：
- 出力先が既に存在するとビルダーはエラーで止まる。`--overwrite` は**自分がこのセッションで生成した雛形を作り直すときだけ**使う。
- ユーザーの完成レポート・提出物のパスを出力先に指定しない。ファイル名にユーザーの学籍番号だけのもの（`24TB4039.docx` 等）や `_report` 付きの既存ファイルは完成レポートの可能性が高い。
- `build()` をライブラリとして呼ぶ場合の引数順は `build(spec, out, template=None, overwrite=False)`。

### 手順5：確認を促す

生成した docx のパスを伝え、Word で開いて確認してもらう。実験前モードなら「赤字部分を実験後に埋めれば完成」と伝える。

## 書式の要点（詳細は report-format.md）

- 表紙＝ＭＳゴシック（題24pt/他16pt）、本文＝ＭＳ明朝。見出し1/2/3＝16/14/12pt太字。
- 見出し番号は手動でテキストに含める（`2.1 PID制御`）。
- **図キャプションは画像の下**、**表キャプションは表の上**、どちらも中央・Caption スタイル。
- 数式は独立行・中央・右に式番号。
- 実験方法は過去形。参考文献は URL のみ不可・Wikipedia 不可。
