# ES 設問の類型と評価観点・書き方の型

Researcher の設問逆算と Writer の執筆方針の共通リファレンス。
設問は表現が違っても、たいてい以下のどれかに分類できる。「人事が何を見たいか」を外さないこと。

## 目次
- 自己PR / 強み
- ガクチカ（学生時代に力を入れたこと）
- 志望動機
- 挫折・困難・失敗経験
- 長所と短所 / 弱み
- 入社後にやりたいこと / キャリアビジョン
- チームでの役割 / リーダーシップ
- 書き方の型（PREP / STAR）

## 自己PR / 強み
- **人事が見たいもの**: 仕事で再現できる強みか／自覚と裏づけがあるか。
- **型**: 結論（強み）→ それを示す具体エピソード（数字つき）→ 仕事でどう活きるか。
- 注意: 「強みは責任感です」だけはNG。行動で証明する。

## ガクチカ（学生時代に力を入れたこと）
- **人事が見たいもの**: 主体性・課題設定力・行動特性の再現性。結果の大小より**プロセス**。
- **型（STAR）**: 状況 → 課題（自分が何を問題と捉えたか）→ 行動（具体的に何をしたか）→ 結果（数字）→ 学び。
- 注意: 凄い実績より「自分で考えて動いた」が伝わるか。役職や肩書きに頼らない。

## 志望動機
- **人事が見たいもの**: 企業理解の深さ／他社では言えない理由／入社後の活躍イメージ／本気度。
- **型**: なぜこの業界 → なぜこの会社（事業・理念・求める人物像と自分の接点）→ 入社後にやりたいこと。
- 注意: 理念をなぞるだけ・どの会社にも出せる内容はNG。ブリーフの「求める人物像」と接続する。

## 挫折・困難・失敗経験
- **人事が見たいもの**: ストレス耐性・課題への向き合い方・そこから学ぶ力。
- **型**: 状況 → 直面した困難（具体的に）→ どう考え行動したか → 結果と学び → 今にどう活きているか。
- 注意: 「困難」だけ書いて「どう乗り越えたか」が薄いと評価されない。両方書く。

## 長所と短所 / 弱み
- **人事が見たいもの**: 自己客観視ができるか／短所への対処をしているか。
- **型**: 短所を正直に → それによる具体的失敗 → 改善のために今していること。
- 注意: 「短所は完璧主義」のような長所の言い換えは見抜かれる。本物の短所＋対処をセットで。

## 入社後にやりたいこと / キャリアビジョン
- **人事が見たいもの**: 企業理解にもとづく現実的なビジョンか／長く活躍してくれそうか。
- **型**: 実現したいこと → なぜこの会社なら可能か（事業・職種と接続）→ そのための強み。

## チームでの役割 / リーダーシップ
- **人事が見たいもの**: 組織の中での立ち回り・協働姿勢。必ずしも「リーダー」でなくてよい。
- **型**: チームの状況 → 自分が担った役割 → 具体的行動 → チームへの貢献（数字や変化）。

## 書き方の型まとめ
- **PREP**: Point（結論）→ Reason（理由）→ Example（具体例）→ Point（結論）。短い設問向き。
- **STAR**: Situation → Task → Action → Result。経験を語る設問向き（ガクチカ・自己PR・挫折）。
- 共通: 結論ファースト・1エピソードに絞る・固有名詞と数字で具体化・設問の全要素に答える。
