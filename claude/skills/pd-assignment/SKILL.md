# pd-assignment

## 前提情報

- 授業: 千葉大学「プログラム設計(pd)」
- 課題ディレクトリ: `~/chiba-u/prog-design/<番号>/`（例: `07` → `~/chiba-u/prog-design/07/`）
- 学生証番号: `24TB4039`
- 前回課題ディレクトリ: `~/chiba-u/prog-design/<番号-1>/`

## ファイル構成

各課題ディレクトリには以下がある:

| ファイル | 説明 |
|---|---|
| `SlidePDxx_2026.pdf` | 講義スライド。課題仕様はここに書いてある |
| `pd0Xa_24tb4000.cpp` | 教員配布スケルトン(a課題) |
| `pd0Xb_24tb4000.cpp` | 教員配布スケルトン(b課題) |
| `pd0Xa_24tb4039.cpp` | **編集対象**(a課題) |
| `pd0Xb_24tb4039.cpp` | **編集対象**(b課題) |

## ワークフロー

### 1. PDF確認

`SlidePDxx_2026.pdf` を読んで課題仕様を把握する。PDFは複数ページあるので `pages` パラメータを分けて読む（最大20ページ/回）。課題仕様ページ（`課題pd0Xa`、`課題pd0Xb`のタイトルがある箇所）を重点的に読む。

### 2. スケルトンと自分のファイルを確認

- 教員スケルトン(`24tb4000`)を読んで main() の呼び出しパターンを把握する
- 自分のファイル(`24tb4039`)を読んで現状を把握する

### 3. 前回のコードスタイルを参照

前回課題ディレクトリの `pd0Xa_24tb4039.cpp` / `pd0Xb_24tb4039.cpp` を読む。以下を合わせる:
- コメントスタイル（日本語、短め）
- インデント・スペーシング
- `std::cout` の使い方

### 4. 実装

**必須ルール:**
- メンバ変数名は `_` プレフィックス（例: `_rows`, `_year`）
- コンストラクタと Set() で処理を重複させない → コンストラクタは対応する Set() を呼ぶだけにする
- 動的メモリ(`new`)を使ったらデストラクタで `delete[]`

**学生らしく書くルール（重要）:**
- 関数名は英語でも短くシンプルに（例: `eraToYear` ○、`eraToWestern` × 、`parseString` × → `setDate` ○）
- `<iomanip>` の `setw/fixed/setprecision` より `printf("%6.2f", ...)` の方が自然
- `struct tm t = {}` より `memset(&t, 0, sizeof(t))` の方が学生らしい
- 過度な抽象化・private helper の細分化を避ける
- AI臭のある命名（`compute`, `process`, `handle`, `manage`, `western`）を避ける

**コメント:**
- ファイル冒頭の「説明:」に工夫した点を箇条書き
- 調べて使った関数があれば「調べた関数:」に記載

### 5. コンパイル確認

```bash
g++ pd0Xa_24tb4039.cpp -Wall -Wextra -std=c++17 -pedantic -o /tmp/pd0Xa
/tmp/pd0Xa
```

バイナリは必ず `/tmp/` に出力してカレントディレクトリを汚さない。

### 6. コミット・プッシュ

```bash
git add pd0Xa_24tb4039.cpp pd0Xb_24tb4039.cpp
git commit -m "finish 0Xa" # または "finish 0Xb"
git push
```

## よくある落とし穴

- 過去の日付の日数差で `+ 0.5` rounding → 負の値は `- 0.5` が正しい
- `localtime()` の戻り値ポインタは静的領域 → 使う前に struct にコピーする: `struct tm t = *localtime(&now);`
- コピーコンストラクタで `_data` をそのままコピーするとシャローコピーになる → `new` して要素を1つずつコピー
