# Plan Handoff

このセッションは「設計役」に徹する。コードは実装せず、(1) 詳細プランを永続ファイルに書き、(2) 別セッションに貼るだけで実装が始まる自己完結プロンプトを pbcopy する。

組み込みの `handoff` スキルとは別物。`handoff` は会話の圧縮引き継ぎ。本スキルは「実装プランを書いて別セッションへ委譲」する用途。

## 手順

### 1. プランを確定する
タスクを調査し、実装に必要な詳細を詰める。プランには必ず含める:
- **Context** — なぜ必要か（意図・背景）と選択理由。
- **手順** — 対象ファイル・行番号・具体的変更を、別セッションが読んだだけで実装できる粒度で。再利用すべき既存関数があればパス付きで示す。
- **検証手順** — 実際に動かして確認する方法（テスト実行・アプリ起動・MCP 等）。
- **完了処理** — 後述の必須項目を含む。

### 2. 永続パスへ保存する
保存先は **対象プロジェクトの直下** に `PLAN-<topic>.md`（例: `~/my-projects/yt-learn/PLAN-issue-27-20.md`）。
- 別セッションは別プロセス。`~/.claude/plans/` のような一時パスではなく、実体のある絶対パスに置く。
- 一時成果物なので `PLAN-` 接頭辞で明示。コミット対象にはしない。
- パスが曖昧なら保存前にユーザーへ確認する。

プラン冒頭に「このプランは別セッションが実装する引き継ぎ文書。完了後はこのファイル自身を削除すること」と1行明記する。

### 3. 完了処理に必ず後片付けを書く
プラン末尾の「完了処理」セクションに必ず含める:
- 実装で必要な最終アクション（テスト緑化・コミット・`gh issue close <N>` など、タスク由来のもの）。
- **本プランファイル自身の削除**（例: `trash <保存した絶対パス>`）。これを忘れると一時ファイルが残る。

### 4. 引き継ぎプロンプトを pbcopy する
別セッションにそのまま貼れる自己完結プロンプトを作り、`pbcopy` に流す。プロンプトには必ず:
- 保存したプランファイルの**絶対パス**と「まず最後まで読む」指示。
- 「記載の全セクションを順に実装し、検証手順まで実施」する指示。
- 後片付け（issue close → プランファイル削除）の指示。
- 注意事項: 破壊的/共有状態への操作（リモートブランチ削除・push・`gh issue close`・force push 等）は実行前にユーザー確認。`rm` は使わず `trash`。プロジェクトの CLAUDE.md に従う。

ヒアドキュメントで pbcopy する（`<<'EOF'` でクォートし変数展開を防ぐ）:
```bash
cat <<'EOF' | pbcopy
<対象プロジェクト> の <タスク概要> を実装してほしい。

詳細プラン: <保存した絶対パス>

まずこのプランファイルを最後まで読み、記載の全セクションを順に実装すること。各変更は対象ファイル・行・手順がプランに明記されている。実装後はプラン末尾の「検証手順」を実施する。

注意:
- 破壊的/共有状態への操作（リモートブランチ削除・push・gh issue close 等）の前は必ずユーザーに最終確認する。
- rm は使わず trash を使う。プロジェクトの CLAUDE.md に従う。
- すべて完了・検証したら完了処理（issue close 等）を行い、最後に本プランファイル自身を trash で削除する。
EOF
echo "copied to clipboard"
```

### 5. ユーザーへ報告する
- 保存したプランファイルの絶対パス。
- 「引き継ぎプロンプトをクリップボードにコピー済み。別セッションで貼れば実装が始まる」。

## 注意
- 本セッションでは実装・コミット・破壊的操作をしない。設計と引き継ぎ準備のみ。
- プランが大きい時はステップ分割し、各セクション完成ごとにファイルへ追記する。
