# IEEE参考文献ジェネレーター

## 目的

IEEE形式の参考文献を**作成**または**整形**する。

- **作成モード**: URL・著者名・タイトル等の情報から参考文献を生成する
- **整形モード**: 崩れたIEEE形式の参考文献を正規化する

---

## 動作モードの判定

| 引数の状態 | 動作 |
|---|---|
| 引数なし | 対話形式で情報を収集して作成 |
| URLや部分的な情報（著者名、タイトル等） | 不足フィールドを質問して作成 |
| 既存の参考文献テキスト（`[N]` 形式 or `Available:` 含む） | 整形モードで正規化 |

---

## 作成モード

### Step 1: メディア種別の確定

引数から HTTP/HTTPS の URL が検出された場合は **Webサイト** と判断して確定する。

それ以外で種別が不明な場合は `AskUserQuestion` ツールを使い、選択肢UIで質問する：

```
question: "参考文献の種類を選んでください"
header: "種別"
options:
  - label: "Webサイト・オンラインリソース"
  - label: "学術論文（ジャーナル）"
  - label: "学会発表（Conference）"
  - label: "書籍"
  - label: "規格書（IEC / ISO / JIS 等）"
```

### Step 2: 不足フィールドの収集

種別ごとの必要フィールドを確認し、**引数に含まれていないフィールドのみ**質問する。
**必ず `AskUserQuestion` ツールを使って**フィールドごとに選択肢または入力を促す。
複数の不足フィールドがある場合は **1回の `AskUserQuestion` 呼び出しにまとめる**（最大4問）。

- 自由入力フィールド（著者名、タイトル等）は `options` に代表的な例や「わからない」を含める
- 「わからない」「不明」「なし」と回答されたフィールドは **空白のまま省略**して出力する

#### Webサイト
| フィールド | 重要度 |
|---|---|
| 著者 / 組織名 | 推奨 |
| タイトル | **必須** |
| URL | **必須** |
| 発行日 | 任意（なければ省略） |
| アクセス日 | 推奨 |

#### 学術論文（ジャーナル）
| フィールド | 重要度 |
|---|---|
| 著者名（複数可） | **必須** |
| 論文タイトル | **必須** |
| 雑誌名 | **必須** |
| vol 番号 | 推奨 |
| no 番号 | 推奨 |
| ページ範囲 | 推奨 |
| 発行年 | **必須** |

#### 学会発表（Conference）
| フィールド | 重要度 |
|---|---|
| 著者名（複数可） | **必須** |
| 論文タイトル | **必須** |
| 学会名 | **必須** |
| 開催地 | 推奨 |
| 発行年 | **必須** |
| ページ範囲 | 推奨 |

#### 書籍
| フィールド | 重要度 |
|---|---|
| 著者名 | **必須** |
| タイトル | **必須** |
| 出版社 | **必須** |
| 発行年 | **必須** |

#### 規格書（IEC / ISO / JIS 等）
| フィールド | 重要度 |
|---|---|
| 規格番号 | **必須** |
| タイトル | **必須** |
| 発行年 | **必須** |

### Step 3: 参考文献の生成

収集した情報を使って出力フォーマット仕様に従って参考文献を生成する。
番号は `[1]` から始める。

生成した参考文献テキストを：
1. チャット上に**コードブロックを使わずプレーンテキストで**表示する（コードブロックはターミナルでインデントが入るため使わない）
2. `Bash` ツールで `printf '%s' "テキスト" | pbcopy` を実行してクリップボードにコピー
3. 「クリップボードにコピーしました」と一言添える

## 整形モード

既存の崩れたIEEE参考文献が渡された場合の処理。

1. 入力の各エントリを `[N]` 単位で分割する
2. 各エントリのメディア種別を判定する（URLあり→Web、vol./no.あり→ジャーナル、学会名あり→Conference、出版社→書籍、規格番号→規格書）
3. 出力フォーマット仕様に従って整形する
4. 日付文字列を `YYYY/M/D` に正規化する
5. 整形後テキストをチャットに**コードブロックを使わずプレーンテキストで**表示し、`printf '%s' "テキスト" | pbcopy` でクリップボードにコピーする

### 表記揺れ吸収ルール

| 入力に含まれるもの | 処理 |
|---|---|
| `[オンライン].` | 削除 |
| `Available:` | 削除（次の URL をそのまま使う） |
| `[アクセス日: 30 4 2026]` | → `アクセス日: 2026/4/30` |
| `[アクセス日: April 30, 2026]` | → `アクセス日: 2026/4/30` |
| `[アクセス日: 2026年4月30日]` | → `アクセス日: 2026/4/30` |
| `4 10 2019`（発行日） | → `2019/4/10` |
| `October 4, 2019`（発行日） | → `2019/10/4` |
| `and` の前のカンマなし | IEEE準拠で `and` の前にカンマを補う |
| 著者名のイニシャルなし | そのまま保持（補完しない） |

---

## 出力フォーマット仕様

### 共通ルール

- `[N]` の直後は**半角スペース1個**
- 著者複数の場合は IEEE 準拠: `A. 山田, B. 鈴木, and C. 田中,`
- 日付は**ゼロ埋めなし**の `YYYY/M/D` 形式（例: `2026/4/30`）
- 文末ピリオドは各エントリの最後に1個つける（URL ブロックがある場合はアクセス日行の末尾）
- タイトルはダブルクォート `"タイトル,"` で囲む（末尾カンマはクォート内）
- 不明・空白のフィールドは出力から省略する（余分なカンマや空白は残さない）

### メディア種別ごとのフォーマット

#### 1. Webサイト・オンラインリソース

```
[N] 著者/組織名, "タイトル," 発行日（あれば）
URL
アクセス日: YYYY/M/D.
```

#### 2. 学術論文（ジャーナル）

```
[N] 著者, "論文タイトル," 雑誌名, vol. X, no. Y, pp. XX–XX, YYYY.
```

#### 3. 学会発表（Conference）

```
[N] 著者, "論文タイトル," 学会名, 開催地, YYYY, pp. XX–XX.
```

#### 4. 書籍

```
[N] 著者, "タイトル," 出版社, YYYY.
```

#### 5. 規格書（IEC / ISO / JIS 等）

```
[N] 規格番号, "タイトル," YYYY.
```

## 使用例

### 例1: URL引数あり（不足情報をダイアログで確認）

```
/mkref https://www.taoglas.com/jp/low-pass-rf-filters/
```

→ `AskUserQuestion` で不足フィールドをまとめて確認（著者名・タイトル・発行日・アクセス日）
→ 回答後に参考文献を表示 + pbcopy

### 例2: 情報なし（種別選択ダイアログから開始）

```
/mkref
```

→ `AskUserQuestion` で種別を選択 → 種別に応じたフィールドをダイアログで確認 → 出力 + pbcopy

### 例3: 整形モード

```
/mkref [1] TAOGLAS, "ローパスRFフィルタ," [オンライン]. Available: https://example.com. [アクセス日: 30 4 2026].
```

→ 整形して表示 + pbcopy:

```
[1] TAOGLAS, "ローパスRFフィルタ,"
https://example.com
アクセス日: 2026/4/30.
```

---

## 注意事項

- 著者名・タイトルの中身は変更しない（誤字も含めてそのまま）
- 番号 `[N]` は入力の順番を維持する（作成モードは `[1]` から始める）
- 「わからない」と回答されたフィールドは省略して出力する（空のカンマ等は残さない）
- 情報が入力に存在しない場合は補完しない
