# IEEE参考文献フォーマッター

## 目的

IEEE形式で書かれた（または書こうとしている）参考文献を、統一された読みやすい形式に整形する。
入力の表記揺れ・余分なラベル・日付フォーマットの乱れを吸収して出力する。

---

## 出力フォーマット仕様

### 共通ルール

- `[N]` の直後は **半角スペース1個**
- 著者複数の場合は IEEE 準拠: `A. 山田, B. 鈴木, and C. 田中,`
- 日付は **ゼロ埋めなし** の `YYYY/M/D` 形式（例: `2026/4/30`、`2019/10/4`）
- 文末ピリオドは各エントリの最後に1個つける（URLブロックがある場合はアクセス日行の末尾）
- タイトルはダブルクォート `"タイトル,"` で囲む（末尾カンマはクォート内）

### メディア種別ごとのフォーマット

#### 1. Webサイト・オンラインリソース

```
[N] 著者/組織名, "タイトル," 発行日（あれば）
URL
アクセス日: YYYY/M/D.
```

- `[オンライン].`, `Available:`, `[アクセス日: DD MM YYYY].` などは削除
- 発行日がない場合はタイトル行に日付を書かない
- 発行日がある場合はタイトルの後に `YYYY/M/D` を追記

#### 2. 学術論文（ジャーナル）

```
[N] 著者, "論文タイトル," 雑誌名, vol. X, no. Y, pp. XX–XX, YYYY.
```

#### 3. 学会発表（Conference）

```
[N] 著者, "論文タイトル," 学会名, 開催地, YYYY, pp. XX–XX.
```

#### 4. 書籍

```
[N] 著者, "タイトル," 出版社, YYYY.
```

#### 5. 規格書（IEC / ISO / JIS 等）

```
[N] 規格番号, "タイトル," YYYY.
```

---

## 入力の吸収ルール（表記揺れ対応）

| 入力に含まれるもの | 処理 |
|---|---|
| `[オンライン].` | 削除 |
| `Available:` | 削除（次のURLをそのまま使う） |
| `[アクセス日: 30 4 2026]` | → `アクセス日: 2026/4/30` に変換 |
| `[アクセス日: April 30, 2026]` | → `アクセス日: 2026/4/30` に変換 |
| `[アクセス日: 2026年4月30日]` | → `アクセス日: 2026/4/30` に変換 |
| `4 10 2019`（発行日） | → `2019/4/10` に変換 |
| `October 4, 2019`（発行日） | → `2019/10/4` に変換 |
| `and` の前のカンマなし | IEEE準拠で `and` の前にカンマを補う |
| 著者名のイニシャルなし | そのまま保持（補完しない） |

---

## 処理手順

1. 入力の各エントリを `[N]` 単位で分割する
2. 各エントリのメディア種別を判定する（URLあり→Web、vol./no.あり→ジャーナル、学会名あり→Conference、ISBN/出版社→書籍、規格番号→規格書）
3. 上記フォーマット仕様に従って整形する
4. 日付文字列を `YYYY/M/D` に正規化する
5. 整形済みリストをまとめて出力する

## 出力例

### 入力（崩れたIEEE形式）

```
[1] 	TAOGLAS, "ローパスRFフィルタ：高周波干渉を排除する鍵," [オンライン]. Available: https://www.taoglas.com/jp/low-pass-rf-filters-the-key-to-eliminating-high-frequency-interference/. [アクセス日: 30 4 2026].
[2] 	おるてぃのひとりごと, "医療工学解説－フィルタ（濾波器）," 4 10 2019. [オンライン]. Available: https://oltyblog.com/ikou-filter. [アクセス日: 30 4 2026].
[3] 	日本真空光学株式会社, "バンドパスフィルター (BPF)," [オンライン]. Available: https://evort.jp/article/band-pass-filter. [アクセス日: 30 4 2026].
```

---

## 注意事項

- 情報が入力に存在しない場合は補完しない（URLが書かれていなければURLは出力しない）
- 著者名・タイトルの中身は変更しない（誤字も含めてそのまま）
- 番号 `[N]` は入力の順番を維持する
